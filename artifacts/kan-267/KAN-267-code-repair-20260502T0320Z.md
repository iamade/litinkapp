## Root Cause Analysis

Two issues caused the audio task execution failure:

### 1. Missing `audio_task_id` persistence (`ai/routes.py`)
- The `/generate-audio-for-script` endpoint dispatched the Celery task via `.delay()` but never persisted the returned `task.id` to `video_generations.audio_task_id`.
- The task ID was only returned in the HTTP response JSON, making it ephemeral.
- When PSQ later checked for audio status, no durable task reference existed in the DB.

### 2. Corrupted code in `generate_all_audio_for_video` (`audio_tasks.py`)
- A 150+ line block of developer commentary from a failed edit attempt was left in the source.
- The commentary contained a `pass` statement followed by duplicate functional code.
- While the `pass` didn't create a dead end (code after it was still reachable), the corrupted commentary made the code unmaintainable and obscured the logic flow.
- The duplicate `# Get video generation data` block re-fetched data already available from the first fetch.

## Changes Made

### Fix 1: `backend/app/api/routes/ai/routes.py`

**`/generate-audio-for-script` endpoint (line ~2772):**
- After `task = generate_all_audio_for_video.delay(video_gen_id)`, added:
  ```python
  video_gen.audio_task_id = task.id
  session.add(video_gen)
  await session.commit()
  ```

**`trigger_task_for_step` helper (line ~5011):**
- For `PipelineStep.AUDIO_GENERATION`, added DB persistence of `audio_task_id` after dispatching the Celery task.

### Fix 2: `backend/app/tasks/audio_tasks.py`

**`generate_all_audio_for_video` task (lines 93-262 → 93-127):**
- Removed 150+ lines of corrupted developer commentary
- Removed duplicate `# Get video generation data` block
- Consolidated into clean code that:
  - Fetches `script_style` from the scripts table
  - Persists `self.request.id` (Celery's own task ID) to `video_gen.audio_task_id`
  - Sets `generation_status = "generating_audio"` and commits

## Dual-Layer Durability

Task IDs are now persisted in two places:
1. **Route layer:** `video_gen.audio_task_id = task.id` (from `.delay()` return)
2. **Task layer:** `video_gen.audio_task_id = self.request.id` (from within the task itself)

This ensures even if the route-level write fails, the task self-identifies when it executes.
